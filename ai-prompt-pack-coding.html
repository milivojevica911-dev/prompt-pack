🧠 Learn Concepts Faster

Use these to deeply understand topics — not just memorize them.

Prompt 01
Explain like I'm 10
Explain [concept, e.g. recursion / APIs / pointers] like I'm a complete beginner. Use a real-world analogy first, then show a simple code example. After that, tell me the most common mistake beginners make with it.
Prompt 02
Socratic learning mode
I want to learn [topic] deeply. Don't just explain it — ask me questions one at a time to guide me to understand it myself. Start easy, get harder. Only give me the answer if I'm stuck after 2 tries.
Prompt 03
Visual mental model
Give me a visual mental model for [concept]. Draw it using ASCII art or describe a physical real-world scene I can picture in my head. The goal is that I can close my eyes and "see" how this works.
Prompt 04
The "why does this exist" prompt
Before [concept/tool/feature] existed, what problem did developers face? Why was this solution invented? What would code look like without it? This helps me understand the motivation, not just the mechanics.
Prompt 05
Compare two things
Compare [concept A] vs [concept B] in a simple table. Show: what each is, when to use each, a 3-line code example of each, and the #1 mistake beginners make when choosing between them.
Prompt 06
Test my understanding
I just learned about [topic]. Give me 5 short quiz questions — mix of multiple choice and "what does this code output" style. Then give me the answers with explanations. Be honest if I get something wrong.
Prompt 07
The "what I need to know first" prompt
I want to learn [topic, e.g. React / machine learning / databases]. What are the 5 prerequisite concepts I must understand first? For each one, give me one resource and estimate how long it takes to learn it.
Prompt 08
30-day learning plan
Create a 30-day study plan to go from zero to building a basic [project type, e.g. web app / REST API / CLI tool] in [language/framework]. I have [X hours per day]. Be specific: what to study each week and what mini-project to build at the end of each week.
Prompt 09
Feynman technique
I'm going to explain [concept] to you in my own words. Tell me what I got right, what I got wrong, what I missed, and what the most important thing I should add is. Here's my explanation: [paste your explanation]
Prompt 10
Roadmap for a specific role
I want to become a [frontend / backend / full-stack / data / mobile] developer. Give me a realistic skill roadmap — not overwhelming, just the 10 most important things to learn in order, with a rough time estimate for each one.
🐛 Write & Debug Code

Don't just get answers — learn WHY the code works.

Prompt 11
Explain this code line by line
Explain this code to me line by line like I'm a beginner. For each line, tell me what it does AND why it's there. If any part could be written differently, show me the alternative. [paste your code here]
Prompt 12
Debug with explanation
My code has a bug. Don't just fix it — explain what the bug is, why it happened, and what I should have written instead. Also tell me how I could have caught this bug myself. [paste your buggy code and describe the problem]
Prompt 13
Write code, teach as you go
Write [specific function/script] in [language]. As you write it, add a comment above every 2-3 lines explaining what that block does and why you chose to do it that way. I want to be able to reproduce this myself next time.
Prompt 14
Code review like a senior dev
Review my code like a senior developer. Look for: bugs, bad practices, readability issues, performance problems, and security issues. Be direct. Rank the issues from most to least important and rewrite the problematic parts. [paste code]
Prompt 15
Simplify my code
My code works but feels messy. Rewrite it to be cleaner and more readable without changing what it does. Then explain the 3 main changes you made and why they're better. [paste code]
Prompt 16
Trace through the code
Walk me through what happens when I call [function/run this code] with the input [specific input]. Trace each step of execution like a debugger — show the value of each variable at each step.
Prompt 17
Error message decoder
I got this error message. Explain exactly what it means in plain English, what most commonly causes it, and give me the 3 most likely fixes. Also tell me what I should google if none of those work. Error: [paste error] My code: [paste relevant code]
Prompt 18
Build a function from scratch
Help me write a [description of function] from scratch. Don't give me the finished code right away — walk me through the logic first, ask me what I think the steps should be, then help me code each step one at a time.
Prompt 19
Write unit tests and explain them
Write unit tests for this function. Include edge cases I might have missed. After writing the tests, explain what each test is checking and why that case matters. [paste function]
Prompt 20
Convert between languages
Convert this [Python / JavaScript / Java] code to [target language]. After converting, highlight the 3 biggest differences in how the two languages handle this kind of code, so I understand the translation — not just the result. [paste code]
🛠 Build Real Projects with AI

Use AI as a co-founder, not just a search engine.

Prompt 21
Break down any project
I want to build [project idea]. I'm a beginner in [language/framework]. Break this down into small, specific tasks I can do one at a time. Start from zero. Make each task completable in under 2 hours. List dependencies between tasks.
Prompt 22
MVP first, features later
What's the absolute minimum version of [project] I need to build first to prove the concept works? List only the features that are truly essential for the MVP. Then list 5 features I can add after. Help me resist over-engineering.
Prompt 23
Tech stack recommendation
I want to build [project description]. I'm a beginner. Recommend the best tech stack for this — not the most advanced one, but the one that's easiest to learn and gets the job done. Explain why you chose each part.
Prompt 24
Stuck on a feature — unstick me
I'm building [project] and I'm stuck trying to implement [feature]. Here's what I've tried: [explain attempts]. Don't give me the full solution — give me the next small step I should try, and explain the concept behind it.
Prompt 25
Design the database schema
Help me design the database for [project]. I need to store [describe the data]. Show me the tables, columns, and relationships. Explain why you structured it that way and what problems this design avoids.
Prompt 26
Plan the file structure
I'm building a [type of app] in [framework]. Show me a clean file/folder structure for this project. Explain what goes in each folder and why. Show me what a professional would do, not just what works.
Prompt 27
Build an API endpoint
Help me build a [GET/POST/PUT/DELETE] API endpoint for [purpose] in [framework]. Show me the full code, explain every line, and show me how to test it. Also tell me what could go wrong and how to handle those errors.
Prompt 28
Add authentication
I need to add user login/signup to my [language/framework] project. What's the simplest and most secure way to do this as a beginner? Walk me through it step by step. What should I absolutely NOT do as a beginner with auth?
Prompt 29
Deploy your project
I finished building [project type] in [language/framework]. Walk me through how to deploy it for free so others can use it. Give me step-by-step instructions and tell me which free platform is best for a beginner project like mine.
Prompt 30
Make your project portfolio-ready
I built [project description]. How do I make it look good for a portfolio? What features should I add to impress a recruiter? What should my README.md say? Give me a template README I can customize.
💼 Land Your First Job

Use AI to prepare smarter, not harder.

Prompt 31
Mock interview — easy level
Do a mock coding interview with me for a junior [language] developer role. Start with easy questions. Ask me one at a time, wait for my answer, then give honest feedback on my solution — correctness, efficiency, and how I'd communicate it in a real interview.
Prompt 32
Explain a concept like in an interview
Ask me to explain [concept] as if I'm in a job interview. After I answer, tell me: what I got right, what I missed, what a strong answer would include, and give me a model answer I can learn from.
Prompt 33
Tell me about yourself — practice
Help me write a strong "tell me about yourself" answer for a junior developer interview. I have [X months/years of self-study], I know [languages/skills], and I built [projects]. Make it confident, honest, and under 90 seconds when spoken.
Prompt 34
Optimize my CV/resume
Review my CV for a junior developer role. Tell me what's weak, what's missing, and what a recruiter notices in the first 10 seconds. Then rewrite my experience section to sound more technical and impactful. [paste your CV]
Prompt 35
Cover letter that doesn't suck
Write a short, authentic cover letter for a junior developer position at [company/type of company]. I have [background]. Make it sound human — not like a template. Focus on what I can do for them, not what I want.
Prompt 36
Crack the system design basics
Explain [system design concept, e.g. load balancing / caching / microservices] at a junior developer level. I don't need to design Netflix — I need to understand it well enough to answer basic questions in an entry-level interview.
Prompt 37
Top 20 interview questions for my stack
Give me the 20 most commonly asked interview questions for a junior [stack, e.g. React / Node / Python / Java] developer role. For each question, give me the key points a strong answer should include — not a full answer, just the skeleton.
Prompt 38
LeetCode problem walkthrough
I'm working on this LeetCode problem: [problem name or description]. Don't give me the solution. Instead, ask me questions to guide my thinking: what data structure could work here? What's the brute force? Can we do better? Push me to figure it out.
Prompt 39
Negotiate your first salary
I just got a job offer as a junior developer in [city/country] for [salary offered]. Is this fair? What's the realistic range for this role? Give me a script I can use to negotiate for more without seeming ungrateful or aggressive.
Prompt 40
LinkedIn summary that gets noticed
Write a LinkedIn "About" section for a self-taught junior developer looking for their first job. I know [skills], built [projects], and I'm looking for [type of role]. Make it punchy, honest, and something a recruiter would actually read.
Prompt 41
Cold email to a company
Write a cold email I can send to [type of company] asking about junior developer openings. I have [background]. Make it short (under 100 words), confident, and end with a clear ask. No cringe, no begging.
Prompt 42
Explain a gap in experience
I'm a self-taught developer with no degree and no professional experience. Help me write an answer to the question "why should we hire you over someone with a CS degree?" that's honest but confident and focuses on what makes me valuable.
Prompt 43
GitHub profile readme
Write a GitHub profile README.md for a junior developer. I know [skills], I've built [projects], and I'm open to [type of work]. Make it look clean and professional — include sections for skills, projects, and how to contact me.
Prompt 44
Behavioral question prep (STAR method)
Help me answer this behavioral interview question using the STAR method: "[question, e.g. Tell me about a time you faced a technical challenge]". I'll give you the raw story and you help me structure it well. My story: [describe what happened]
Prompt 45
What to study this week for interviews
I have a technical interview in [X weeks] for a junior [type] developer role. Based on what companies typically test, give me a prioritized study plan for the time I have. I'm comfortable with [topics] but weak on [topics].
Prompt 46
Questions to ask the interviewer
Give me 7 smart questions to ask at the end of a junior developer interview. Mix of questions about the team, the tech stack, growth opportunities, and day-to-day work. Questions that show I'm serious — not just "what does your company do".
Prompt 47
Take-home project tips
I got a take-home coding assignment: [describe the task]. Before I start coding, what are the 5 most important things I should focus on to impress the reviewers? What do most candidates mess up? What would make mine stand out?
Prompt 48
Freelance rate calculator
I'm a junior developer in [country] looking to do freelance work. Help me figure out what hourly rate to charge. Consider local cost of living, international market rates, and my skill level. Give me a range for local clients and international clients separately.
Prompt 49
Network on LinkedIn without being cringe
Write a short LinkedIn connection request message I can send to [senior developers / recruiters / people who work at companies I want to join]. I want to build a genuine connection, not spam them. Make it personal-feeling, under 3 sentences.
Prompt 50
Stay motivated — accountability plan
I'm learning to code and often lose motivation after a few weeks. Create a realistic weekly schedule for me based on [hours per day available] that balances learning, building, and rest. Include milestones I can celebrate to stay motivated long-term.
